# YNC 奖金系数方案测算工具

这是一个基于 Python + Streamlit + Pandas + OpenPyXL/XlsxWriter 的本地 Web 工具，用于 YNC 项目的奖金系数方案测算与对比分析。

## 安装依赖

```bash
pip install -r requirements.txt
```

## 启动工具

```bash
streamlit run app.py
```

启动后，浏览器会打开本地页面。工具不需要数据库、登录或外部 API。

## 客户需要填写的字段

客户只需要填写模板中的 `01_员工数据` Sheet。核心字段包括：

- `employee_id`：员工编号，必填且不可重复
- `employee_name`：姓名，必填
- `department`：部门，建议填写
- `original_job_family`：原职群，必填
- `original_qualification`：原能力资格，必填
- `new_job_family`：新职群，必填，填写 `M/T/S/O/G`
- `new_grade`：新等级，必填，填写实际单一等级，例如 `M4`、`T1`、`S3`、`O4`、`G6`
- `rating`：评价等级，必填
- `bonus_base`：月度奖金基数，必填。上传表头可使用“月度奖金基数”“月奖金基数”或“奖金基数”
- `proration_factor`：折算系数，不填默认 1
- `eligible`：是否参与测算，必填，填写 `Y` 或 `N`
- `remarks`：备注，可选

## 使用流程

1. 在“模板下载”页面下载标准员工数据模板。
2. 客户填写 `01_员工数据` Sheet 后上传。
3. 在“数据上传”页面上传员工数据。
4. 在“数据检查”页面查看异常清单，并确认新职群和新等级是否为有效组合。
5. 在“方案系数设置”页面调整方案A和方案B的奖金系数。
6. 点击“重新测算”，系统刷新奖金明细和汇总结果。
7. 在“对比分析”页面查看总体、职群、等级、评价等级和员工影响清单。
8. 在“结果导出”页面导出完整 Excel 文件。

## 系数配置说明

当前版本在代码中内置了示例系数矩阵：

- 现行方案：按原职群 × 原能力资格 × 评价等级匹配
- 方案A：按新职群 × 评价等级匹配
- 方案B：按新职群 × 新等级 × 评价等级匹配

方案B使用实际单一新等级作为计算键，例如 `M4`、`M5`、`T1`、`S1`、`O1`、`G1`。咨询方案中如有 `M7-8`、`S1-2` 等组标签，仅用于初始化默认相同系数，不作为匹配键。

如果后续拿到完整真实系数，可以替换 `app.py` 中的默认系数生成函数。
